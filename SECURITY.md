# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.0.0-beta.x | Yes |
| < 1.0.0-beta | No |

## Reporting a Vulnerability

Please report security issues privately via **GitHub Security Advisory**.
Do not open public issues for security vulnerabilities.

Include:

- Affected version
- Reproduction steps
- Impact assessment
- Suggested mitigation (if any)

## Disclosure Timeline

We follow a best-effort process:

- Initial acknowledgment: within 5 business days
- Triage and fix planning: as soon as reproducible
- Coordinated disclosure: after patch availability
